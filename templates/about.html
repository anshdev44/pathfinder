Project Overview

This project aims to visualize and compare different path finding algorithms. By providing an interactive interface integrated with real-world maps, users can see how algorithms like Dijkstra's and A* navigate between points, calculating the most efficient routes.

Algorithms Explained

Dijkstra's Algorithm

Click to learn more

Dijkstra's Algorithm

Finds the shortest path between nodes in a graph. It starts at the source node and iteratively selects the unvisited node with the smallest distance, updating the tentative distances to all unvisited neighbors.

  • Time Complexity: O((V+E)logV)
  • Space Complexity: O(V)
  • Optimal for: Finding shortest paths when all edges have non-negative weights
  • Fun Fact: Named after its creator, Dutch computer scientist Edsger W. Dijkstra

A* Algorithm

Click to learn more

A* Algorithm

A* is an informed search algorithm that uses a heuristic function to guide its path exploration. It intelligently evaluates nodes by combining the cost to reach a node and the estimated cost to reach the goal.

  • Time Complexity: O(E)
  • Space Complexity: O(V)
  • Optimal for: Finding shortest paths with additional information about the goal
  • Fun Fact: A* is widely used in video games for pathfinding characters through complex terrain
